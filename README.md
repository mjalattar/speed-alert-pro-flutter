# Speed Alert Pro (Flutter)

Cross-platform port of **`speed-alert-pro-android`** (Kotlin). The original Android project folder is unchanged; this is a parallel implementation for **iOS and Android** using **Flutter / Dart**.

## Prerequisites

- [Flutter](https://docs.flutter.dev/get-started/install) 3.24+ (stable) with `flutter` and `dart` on your `PATH`.

## First-time setup (generate `android/` and `ios/`)

This repository contains the Dart app (`lib/`) and `pubspec.yaml`. Platform folders are generated by Flutter so you always match your installed SDK.

From this directory:

```powershell
flutter create . --project-name speed_alert_pro --org com.speedalertpro
```

Flutter merges in missing files without replacing your `lib/` or `pubspec.yaml`. Then:

```powershell
flutter pub get
```

Create `android/local.properties` if needed (Flutter may do this automatically):

- `sdk.dir` — Android SDK path  
- `flutter.sdk` — Flutter SDK root  

## Run (with HERE API key)

The Android app used `local.properties` / `BuildConfig`; this port uses **`--dart-define`** for keys:

```powershell
flutter run `
  --dart-define=HERE_API_KEY=your_here_rest_key
```

Optional (see **`supabase/SETUP.txt`** in this repo — same backend as the Kotlin app):

```powershell
flutter run `
  --dart-define=HERE_API_KEY=your_here_key `
  --dart-define=SUPABASE_URL=https://YOUR_REF.supabase.co `
  --dart-define=SUPABASE_ANON_KEY=your_anon_key `
  --dart-define=REVENUECAT_PUBLIC_API_KEY=your_rc_public_key `
  --dart-define=GOOGLE_MAPS_API_KEY=your_maps_key `
  --dart-define=GOOGLE_WEB_CLIENT_ID=your_web_client_id.apps.googleusercontent.com
```

When `SUPABASE_URL` is non-empty, `AppConfig.useRemoteHere` is true.

### Auth + subscription (matches Kotlin `MainActivity`)

1. **`AppRootGate`** (`lib/widgets/app_root_gate.dart`): if `SUPABASE_URL` is set, the user must **Sign in with Google** (`GoogleSignInScreen` + `GoogleAuthService`) before the main UI. This mirrors `SupabaseGoogleSignInGate`.
2. **Entitlement**: if Settings → **Use Supabase Edge for speed** is on, the app loads `profiles.trial_ends_at` via Supabase REST and then **RevenueCat** entitlement `premium` — same idea as `EntitlementRepository`. If not trial and not subscribed, **`SubscriptionPaywallScreen`** is shown.
3. **RevenueCat**: after Google sign-in, `Purchases.logIn(supabaseUserId)` runs (also mirrored in Kotlin `syncRevenueCatWithSupabaseUser`).

Use the **Web OAuth client ID** (not the Android-only client id) for `GOOGLE_WEB_CLIENT_ID`. For **iOS**, add the URL scheme from Google’s iOS client (reversed client id) to `Info.plist` after `flutter create` — see [Flutter Google sign-in iOS](https://pub.dev/packages/google_sign_in).

`google_sign_in` 6.x does not pass a hashed nonce like the Android Credential Manager flow; if your Supabase project enforces Google nonce, you may need a newer sign-in stack or a custom channel.

## Parity notes (Kotlin → Flutter)

| Area | Status |
|------|--------|
| HERE Routing v8 alert lead (~1 km) + mph from spans | Implemented in `HereRoutingClient` |
| Supabase Edge `here-speed` | Implemented in `HereEdgeClient` |
| Google gate + trial REST + RevenueCat paywall | `AppRootGate`, `EntitlementRepository`, `SubscriptionPaywallScreen` |
| Preferences / alert threshold / run modes / theme | `PreferencesService` + Settings UI |
| `LocationProcessor` sticky section walk, stabilizers, TomTom/Mapbox compare | Not ported — use Kotlin app or extend Flutter |
| Foreground service, overlay HUD, mock route simulation | Roadmap (plugins / platform channels) |

## Project layout

- `supabase/` — SQL migration, Edge Function `here-speed` (`functions/here-speed/index.ts`), `config.toml`, **SETUP.txt** (deploy from this folder with Supabase CLI)
- `lib/config/` — `AppConfig` (`String.fromEnvironment`, mirrors `BuildConfig` + `local.properties`)
- `lib/services/` — HERE REST, Edge client, preferences, `SpeedLimitCoordinator`, `EntitlementRepository`, `GoogleAuthService`
- `lib/providers/` — Riverpod: prefs, auth session stream, entitlement future, driving session
- `lib/widgets/` — `AppRootGate` (Google + paywall)
- `lib/screens/` — Home, Settings, Google sign-in, subscription paywall
